Chapitre 15 : Styles par défaut et personnalisation

Comprendre les styles par défaut

Les navigateurs appliquent des styles par défaut aux éléments HTML, ce qui peut entraîner des différences d'affichage entre navigateurs.

  • Les styles par défaut varient selon les navigateurs et versions.
  • Ils affectent notamment les marges, paddings, polices, et tailles des éléments.
  • Il est conseillé d'utiliser des "reset CSS" ou "normalize.css" pour uniformiser les styles de base.

Personnaliser vos styles

  • Définissez vos propres règles CSS pour contrôler précisément l'apparence.
  • Utilisez des variables CSS pour gérer facilement les couleurs et tailles.
  • Testez vos styles sur plusieurs navigateurs et appareils.
  • Gardez une cohérence visuelle en adoptant une charte graphique claire.

Exemples de styles par défaut et personnalisations ciblées

Boutons

button {
  all: unset; /* supprime tous les styles natifs */
  cursor: pointer;
  padding: 0.8em 1.2em;
  background-color: var(--primary-color);
  border-radius: 6px;
  color: white;
  transition: background 0.3s;
}
button:hover {
  background-color: var(--primary-hover);
}

Champs (input, textarea)

input, textarea {
  border: 1px solid #ccc;
  padding: 0.6em;
  outline: none;
  transition: border-color 0.2s;
}
input:focus, textarea:focus {
  border-color: var(--primary-color);
  background-color: #f9f9ff;
}

Liens

a {
  color: var(--link-color);
  text-decoration: none;
}
a:hover {
  color: var(--link-hover);
  text-decoration: underline;
}

Neutralisation des styles natifs (reset CSS de base)

*, 
*::before, 
*::after {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
html {
  scroll-behavior: smooth;
}
body {
  line-height: 1.5;
  font-family: system-ui, sans-serif;
}
