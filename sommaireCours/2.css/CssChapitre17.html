Chapitre 17 : Style des formulaires

Pourquoi styliser les formulaires ?

Les formulaires sont essentiels pour l’interaction utilisateur. Un bon style améliore l’accessibilité, la clarté et l’expérience globale.

Éléments de formulaire courants

  • Champs texte, zones de texte, boutons radio, cases à cocher
  • Listes déroulantes et boutons de soumission
  • Labels associés pour améliorer l’accessibilité

Techniques de style

Champs texte

input, textarea {
  padding: 0.8em;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-size: 1rem;
  transition: border-color 0.2s;
}
input:focus, textarea:focus {
  border-color: var(--accent-color);
  outline: none;
}
input::placeholder {
  color: #999;
  font-style: italic;
}

Radios & cases à cocher

On peut masquer l’élément natif pour recréer une apparence personnalisée, ou utiliser la propriété simple accent-color :

input[type="checkbox"],
input[type="radio"] {
  accent-color: var(--accent-color); /* simple */
}

/* Apparence personnalisée (exemple case à cocher) */
input[type="checkbox"] {
  appearance: none;
  width: 1.2em;
  height: 1.2em;
  border: 2px solid #888;
  border-radius: 4px;
}
input[type="checkbox"]:checked {
  background-color: var(--accent-color);
}

Listes déroulantes

On utilise appearance: none; pour supprimer le style natif, puis on ajoute une flèche personnalisée avec une image de fond :

select {
  appearance: none;
  background: white url('chevron.svg') no-repeat right 1em center;
  padding: 0.8em 2.5em 0.8em 1em;
  border-radius: 6px;
}

Curseur input[type=range]

input[type="range"] {
  width: 100%;
  accent-color: var(--accent-color); /* simple */
}
/* Pour personnalisation avancée, utiliser ::-webkit-slider-thumb, ::-moz-range-thumb, etc.

Gestion des erreurs

input:invalid {
  border-color: crimson;
  background-color: #ffe5e5;
}
input:invalid::placeholder {
  color: crimson;
}

Afficher les messages d'erreur accessibles avec aria-live="polite" pour informer l'utilisateur en temps réel.

Accessibilité

Veillez à associer correctement les labels aux champs, utiliser des éléments natifs, et tester la navigation au clavier pour garantir une bonne expérience à tous les utilisateurs.
