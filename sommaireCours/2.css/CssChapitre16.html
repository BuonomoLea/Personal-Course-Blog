Chapitre 16 : Style des SVG et icons

Introduction aux SVG et icônes

Le SVG (Scalable Vector Graphics) est un format vectoriel permettant des images qui conservent leur qualité quel que soit le zoom.

Les icônes SVG sont largement utilisées pour leur flexibilité et légèreté.

Techniques de style des SVG

  • Utiliser les propriétés CSS comme fill, stroke, stroke-width pour personnaliser les formes.
  • Appliquer des transitions et animations CSS aux éléments SVG.
  • Intégrer les SVG inline dans le HTML pour un contrôle direct du style.
  • Utiliser des classes et ID pour cibler précisément des parties du SVG.
svg {
  width: 24px;
  height: 24px;
  fill: currentColor;
}

currentColor permet d’hériter de la couleur du texte ou du parent, facilitant ainsi la gestion des couleurs.

Icônes SVG vs Fonts d'icônes

Les fonts d’icônes sont pratiques mais moins flexibles que les SVG.

Les SVG offrent un rendu plus net et une meilleure personnalisation.

Intégration et librairies d’icônes populaires

  • En <img> : facile à intégrer mais impossible de styliser avec CSS (la propriété fill est ignorée).
  • En inline <svg> : recommandé pour styliser facilement les couleurs et animations.
  • En background : utile pour les icônes purement décoratives.

Librairies populaires

  • Font Awesome — classique, disponible via CDN ou npm.
  • Remix Icons — légères, modernes, facile à utiliser.
  • Heroicons — SVG pur, très utilisés avec Tailwind.
  • Feather, Phosphor, Tabler — icônes très épurées.
  • Iconify — énorme catalogue et système d’unification.

Exemple avec Remix Icons :

<link href="https://cdn.jsdelivr.net/npm/remixicon/fonts/remixicon.css" rel="stylesheet">
