Chapitre 6 : Attributs globaux essentiels

Qu'est-ce qu'un attribut global ?

Un attribut global est un attribut qui peut être appliqué à n’importe quel élément HTML. Ils permettent d’enrichir le comportement ou l’apparence d’un élément sans dépendre de sa nature spécifique.

Les attributs les plus utilisés

  • id : identifiant unique pour un élément
  • class : pour grouper des éléments et leur appliquer des styles
  • style : permet d’appliquer du CSS en ligne
  • title : affiche une info-bulle au survol
  • hidden : masque un élément sans le supprimer du DOM
  • tabindex : contrôle l’ordre de navigation au clavier
  • lang : indique la langue d’un élément particulier
  • data-* : permet de stocker des données personnalisées (utiles en JS)

Tableau récapitulatif des attributs globaux essentiels

Attribut Rôle principal Exemple
class Associer un élément à un ou plusieurs styles CSS <div class="carte">
id Identifier de façon unique un élément <section id="contact">
style Ajouter un style CSS directement (à éviter) <p style="color: red;">
title Info-bulle au survol de l’élément <img title="Logo principal">
lang Indique la langue de l’élément <html lang="fr">
tabindex Ordre de tabulation pour la navigation clavier <button tabindex="0">
hidden Masquer un élément (même s’il reste dans le DOM) <div hidden>
aria-* Attributs pour l’accessibilité (ARIA) <button aria-label="Fermer">
data-* Attributs personnalisés pour stocker des données <div data-prix="19.99">
role Donne un rôle ARIA sémantique (accessibilité) <nav role="navigation">
contenteditable Rendre un élément modifiable par l’utilisateur <div contenteditable="true">
draggable Rendre un élément déplaçable <img draggable="true">

Exemple pratique


<p id="intro" class="highlight" title="Texte introductif">
    Bienvenue sur notre site !
</p>

<div data-role="carte" hidden>...</div>

Ici, plusieurs attributs globaux sont utilisés pour gérer l’apparence, l’accessibilité et les interactions JavaScript.

Bonnes pratiques

  • Éviter les styles en ligne sauf cas exceptionnel
  • Préférer les classes pour le style et les IDs pour les interactions ciblées
  • Utiliser les attributs data-* pour transmettre des infos dynamiques à JavaScript
  • Bien documenter l’usage des attributs personnalisés
