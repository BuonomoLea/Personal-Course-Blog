Chapitre 14 : Organisation et bonnes pratiques

Organisation des fichiers CSS

Il est essentiel de structurer vos fichiers CSS pour faciliter la maintenance et la collaboration :

  • Séparez les styles globaux, les composants, et les pages spécifiques dans des fichiers distincts.
  • Utilisez une convention de nommage claire pour les fichiers (ex : base.css, header.css, footer.css).
  • Regroupez vos fichiers dans des dossiers logiques (ex : /CSS/components/, /CSS/layout/).

Bonnes pratiques CSS

  • Utilisez des préprocesseurs comme Sass ou Less pour modulariser votre CSS.
  • Favorisez l’écriture de règles spécifiques et évitez les sélecteurs trop génériques.
  • Commentez votre code pour expliquer les choix complexes.
  • Optimisez la performance en minimisant et en regroupant les fichiers CSS en production.
  • Testez régulièrement vos styles sur différents navigateurs et appareils.

Structuration logique du fichier CSS

Une organisation logique facilite la lecture et la maintenance :

  1. Variables (définies dans :root pour couleurs, espacements, etc.)
  2. Reset / normalisation des styles de base
  3. Styles généraux (typographie, styles globaux sur body, html)
  4. Layout (grilles, containers, sections)
  5. Composants (boutons, cartes, menus...)
  6. Utilitaires (classes helpers, helpers rapides)
  7. Media queries (organisées par breakpoint ou en blocs regroupés)

Convention BEM & commentaires

Utilisez la méthodologie BEM pour nommer vos classes :

/* === COMPOSANT : Carte === */
.card {}
.card__image {}
.card__title {}
.card--highlight {}

Approche modulaire avec SCSS

Divisez votre CSS en fichiers séparés, puis importez-les :

@import "variables";
@import "reset";
@import "layout";
@import "buttons";
@import "forms";

SCSS permet aussi l’imbrication, les boucles, mixins, etc., pour un code plus maintenable.
