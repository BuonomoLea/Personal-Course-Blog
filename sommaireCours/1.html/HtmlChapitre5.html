Chapitre 5 : Les classes et ID

Class

Sert à cibler plusieurs éléments à la fois.

Peut être utilisée autant de fois que nécessaire dans la page.

Se note avec un point en CSS : .menu, .btn

Exemple :

<div class="container">Contenu</div>

.container {
  propriété: valeur;
}

ID

Sert à identifier un élément unique dans la page.

Ne doit être utilisé qu'une seule fois par page.

Se note avec un dièse (#) en CSS : #header, #formulaire-contact

Exemple :

<h1 id="title">Bienvenue</h1>

#title {
  color: yellow;
}

Utilisation recommandée

Utiliser des classes pour tout ce qui est répétable ou générique

Réserver les id pour des éléments uniques (ancres, ciblage JS…)

BEM : Block — Element — Modifier

BEM est une méthode de nommage des classes CSS qui suit une logique simple et prévisible. Voici les 3 composants :

  • Block : Représente un composant autonome de l’interface.
  • Element : Partie d’un bloc, dépendante de celui-ci. On le note avec deux underscores __.
  • Modifier : Variation d’apparence ou d’état du bloc ou de l’élément. On le note avec deux tirets --.

Exemple :

<article class="article"> <!-- Block -->
  <h2 class="article__titre">Titre de l’article</h2> <!-- Element -->
  <p class="article__contenu">Texte descriptif...</p> <!-- Element -->
  <a class="article__bouton article__bouton--important" href="#">Lire plus</a>
</article>  <!-- Modifier -->

Classes génériques et bien nommées

  • .container — Conteneur centralisé avec largeur max
  • .row — Ligne de grille (souvent utilisée avec Flex/Grid)
  • .col, .col-6, .col-12 — Colonnes responsives
  • .flex, .flex-row, .flex-column — Disposition en flexbox
  • .grid, .grid-2, .grid-3 — Mise en grille
  • .align-center, .justify-center — Alignement horizontal/vertical

Blocs de contenu

  • .section — Section visuelle du site
  • .card — Bloc de type “fiche produit” ou “article”
  • .card__header, .card__body, .card__footer — Sous-éléments BEM
  • .hero — Grande bannière d’intro
  • .wrapper — Conteneur générique

Texte et typographie

  • .title, .subtitle — Titres secondaires
  • .text-center, .text-right, .text-left — Alignement du texte
  • .text-muted — Texte moins visible
  • .uppercase, .lowercase, .capitalize — Transformations du texte
  • .lead — Paragraphe important/mis en avant

Formulaires

  • .form-group — Groupe de champ/label/input
  • .form-control — Champ de texte
  • .form-label — Étiquette du champ
  • .input-error — Champ avec erreur visuelle
  • .input-valid — Champ validé

Boutons & interactions

  • .btn — Bouton de base
  • .btn-primary, .btn-secondary, .btn-danger — Variantes colorées
  • .btn-block — Bouton pleine largeur
  • .btn-sm, .btn-lg — Tailles

Espacements

  • .m-0, .mt-2, .mb-4 — Marges (top, bottom, etc.)
  • .p-0, .pt-2, .px-3 — Paddings
  • .gap-1, .gap-2 — Espacement entre éléments en grille ou flex

Couleurs et arrière-plan

  • .bg-light, .bg-dark, .bg-primary — Arrière-plans
  • .text-primary, .text-danger, .text-success — Couleurs de texte

Résumé

  • Utiliser des classes génériques et bien nommées (.btn, .container, .card, .form-group)
  • Composants modulaires : chaque bloc HTML représente une "brique" réutilisable (ex. carte produit, bloc d’alerte)
  • Approche BEM : structure ton CSS de manière prévisible pour pouvoir dupliquer facilement un composant
  • Variables CSS (--couleur-primaire, --font-base) : pour uniformiser les couleurs, tailles, etc.
  • Éviter les styles en ligne (style="color:red") → privilégie les feuilles de style
  • Responsivité : classes utilitaires pour le responsive (.col-12, .flex-md-column)
